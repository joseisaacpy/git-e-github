Principais Comandos e Conceitos

Veja o curso!

  • git init

    Inicializar um repositório Git.

  • git status

    Verificar o status do repositório.

  • git add .

    Adicionar todos os arquivos para o stage, preparando-os para serem commitados.

  • git add `nome do arquivo`

    Adicionar um único arquivo para o stage, preparando-o para ser commitado.

  • git commit -m `mensagem`

    Commitar os arquivos do stage com uma mensagem explicativa.

  • git config --global user.email "seuemail@example.com"

    Definir o e-mail globalmente para que o Git não peça novamente.

  • git config --global user.name "Seu Nome"

    Definir o nome de usuário globalmente no Git.

  • git origin add `link do repositório`

    Adicionar um repositório remoto para sincronizar com o repositório local.

  • git push origin master

    Enviar os commits para o repositório remoto.

  • git pull origin master

    Atualizar o repositório local com os commits do repositório remoto.

  • git branch -a

    Verificar todos os branches do repositório local.

  • git log

    Exibe o histórico oficial de commits do repositório.

  • git reflog

    Mostra todas as movimentações do HEAD, incluindo commits "perdidos" ou fora da linha do tempo principal.

  • git reset --soft <commit>

    Move o HEAD para o commit indicado, mantendo as alterações na staging area e no diretório de trabalho.

  • git reset --mixed <commit>

    Move o HEAD para o commit indicado e limpa a staging area, mas mantém as alterações no diretório de trabalho.

  • git reset --hard <commit>

    Move o HEAD para o commit indicado e descarta tanto a staging area quanto as alterações no diretório de trabalho.

  • git branch `nome da branch` ou git branch

    Cria uma nova branch com o nome escolhido, mas não muda de branch automaticamente. No outro caso, mostra as branches existentes.

  • git checkout `nome da branch`

    Muda de branch para a escolhida.

  • git checkout -b `nome da branch`

    Cria uma nova branch e muda de branch para ela.
